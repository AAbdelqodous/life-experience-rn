# Implementation Plan: Phase 4.0 — Deep Trust

**Branch**: `004-phase-4-deep-trust` | **Date**: 2026-04-15 | **Spec**: [spec.md](spec.md)
**Input**: Feature specification from `/specs/004-phase-4-deep-trust/spec.md`

## Summary

Phase 4.0 adds four deep-trust features to the existing React Native app: a live work progress timeline on booking detail, a work photos gallery for bookings, a quote review & approval flow, and trust badges on center detail. All features require new backend endpoints not yet implemented. Three new RTK Query slices are introduced (progressApi, mediaApi, quoteApi); the existing centersApi is extended with a badges endpoint; and `BookingStatus` gains a `QUOTE_READY` value. Two new screens are created (progress, photos) and two existing screens are extended (bookings/[id], centers/[id]).

## Design Decisions

### WorkProgressTimeline — Data Ownership
The `progress.tsx` screen owns the `useGetBookingProgressQuery` call and passes the full `WorkProgressResponse` to `WorkProgressTimeline`. The component is purely presentational. This keeps the loading/error state in one place.

### WorkProgressStep — Connector Line
Each `WorkProgressStep` renders a vertical connector line below itself (except the last item). The line is grey for future steps, blue for completed, and pulsing amber for the current step. The `isLast` prop controls visibility.

### QuoteCard — Reject Confirmation Pattern
Rejection shows an inline `TextInput` for the optional reason + a "Confirm Rejection" button — no modal. This mirrors the existing cancel booking dialog inline approach in `bookings/[id].tsx` and avoids nested modals.

### PhotoGrid — Full-Screen Modal
Built-in React Native `Modal` with `expo-image` inside. The modal is managed via `useState({ visible, selectedMedia })` in the `PhotoGrid` component. No navigation push — modal opens on top of the current screen.

### TrustBadgeList — Conditional Render
The `TrustBadgeList` component renders `null` when `badges.length === 0`. The parent screen (`centers/[id].tsx`) calls `useGetCenterBadgesQuery` and only renders the section when `data && data.length > 0`.

### BookingStatus.QUOTE_READY — Color
`#FF6F00` (deep orange-amber) in `getStatusColor()` in `bookings/[id].tsx`. Distinct from PENDING (`#FF9800`) and CONFIRMED (`#2196F3`). Signals a required customer action.

### Navigation to Progress/Photos Screens
Both screens use `useLocalSearchParams()` to read `id` (the booking ID). They are siblings of `bookings/[id].tsx` under `app/(app)/(tabs)/bookings/`. Navigation: `router.push({ pathname: '/(app)/(tabs)/bookings/progress', params: { id: String(bookingId) } })`.

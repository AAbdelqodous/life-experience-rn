# Implementation Plan: Phase 4.5 — Retention

**Branch**: `005-phase-4-5-retention` | **Date**: 2026-04-15 | **Spec**: [spec.md](spec.md)
**Input**: Feature specification from `/specs/005-phase-4-5-retention/spec.md`

---

## Summary

Phase 4.5 adds six customer-retention features to the React Native app: a Loyalty Points & Tier Dashboard, a Rewards Catalog with inline redemption, a Points Transaction History, Vehicle Management with dashboards, Maintenance Reminders with completion gestures, and a Referral Program with native share. All 11 backend endpoints are new. Four new RTK Query slices are introduced. Six new components and eight new screens are created. The Profile screen gains three navigation rows linking to the new features.

---

## Technical Context

**Language/Version**: TypeScript 5.x / React Native 0.81.5 + Expo SDK 54
**Primary Dependencies**: RTK Query (existing), react-i18next (existing), React Hook Form + Zod (existing — vehicle form), Share (built-in react-native), Clipboard (built-in react-native), expo-image (existing), @expo/vector-icons Ionicons (existing)
**Storage**: Redux in-memory store + expo-secure-store (JWT) — no new storage
**Testing**: Jest + @testing-library/react-native (existing)
**Target Platform**: iOS + Android (Expo managed workflow); web via react-native-web
**Performance Goals**: Loyalty dashboard < 1s; photo-free screens render in < 16ms; paginated history loads next page in < 1s
**Constraints**: RTL mandatory; offline-graceful (all screens show empty/cached states); JWT required; vehicle form validates year client-side + server-side
**Scale/Scope**: 4 new API slices, 6 new components, 8 new screens, 1 existing screen updated (profile/index.tsx)

## Design Decisions

### Loyalty Dashboard Layout
Points balance shown in a large hero card (blue background, white text). TierBadge inline next to tier label. PointsProgressBar below hero. ExpiringPoints warning in amber below progress bar (conditionally rendered). Two buttons at bottom: "View Rewards" and "History".

### RewardCard — Inline Redemption Code
After `redeemReward` succeeds, the card state changes: "Redeem" button is replaced by a code display box (bordered `#E8F5E9` background, `#1B5E20` text) with a Copy icon button. `Clipboard.setStringAsync(code)` is called. The component manages its own `redemptionResult` state.

### TierBadge — Color Mapping
BRONZE `#CD7F32`, SILVER `#A8A9AD`, GOLD `#FFD700`, PLATINUM `#E5E4E2` — locked per spec. Text color is always dark (`#1A1A2E`) for contrast. For GOLD specifically, use `#7B6300` text for readability against the yellow background.

### PointsProgressBar — RTL Direction
Progress bar fill goes left-to-right in LTR and right-to-left in RTL. Achieved by `scaleX(-1)` transform on the track view in RTL mode.

### Vehicles Route Namespace
`app/(app)/vehicles/` is a new flat route group (like `complaints/` and `reviews/`). These are not under `(tabs)` since vehicles don't need bottom tab presence.

### Reminders Grouping
`vehicles/reminders.tsx` receives `vehicleId` via `useLocalSearchParams()`. Reminders are grouped client-side: `filter(r => r.status === 'OVERDUE')`, `filter(r => r.status === 'DUE_SOON')`, `filter(r => r.status === 'UPCOMING')`. Each group renders a section header only if it has items.

### Share on Web
`Share.share()` is not supported on web. Use `Platform.OS === 'web'` guard: copy the referral link to clipboard instead of opening the share sheet. Show a toast/alert "Link copied to clipboard".

### Pagination — Points History
`loyalty-history.tsx` owns `const [page, setPage] = useState(0)`. The query is called with `{ page, size: 20 }`. When new page loads, results are accumulated in local state: `const [allTransactions, setAllTransactions] = useState<LoyaltyTransaction[]>([])`. Append on successful query. "Load more" button visible when `!data?.last`.
